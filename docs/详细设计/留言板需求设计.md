## 0. 已确认规则（问答沉淀）

### 0.1 统计用「自然日」

- 采用 **北京时间（UTC+8）**。
- 每个自然日为当日 **00:00:00**（北京时间）起至 **次日 00:00:00**（北京时间）止；限额在该窗口内累计与重置（实现上可用 `[day_start, day_next)` 半开区间存 ISO 时间戳）。

### 0.2 公开展示与审核策略

- **所有访客**均可进入留言板：先浏览 **主题（版块）列表**，再进入某一主题下查看 **已上墙留言**。
- **未具备互动资格**（未完成 **标准或 PRO** 测验，见 **§0.6**）的用户：**仅可观看**，**不可提交留言**；提交入口置灰或隐藏，并提示先完成测试。
- 留言 **必须归属某一主题**（§0.8）；主题由 **管理员** 创建与维护。
- 列表中 **仅展示站主已审核、并选用通过的留言**；未审核或拒绝的留言对公众 **不可见**。
- **顶栏导航**：在首页等页面顶部导航中增加 **「留言板」** 入口（与现有「首页 / PRO 鉴定 / 我的B站」并列），进入留言板页面；视觉需与站点主色（深蓝底、白字、浅紫强调）一致。

### 0.3 站主 / 管理员能力（已确认）

- 采用 **站内极简管理页**（同一套 **环境变量密钥** 鉴权）：站主可进行 **主题（版块）管理** 与 **留言审核、调权**，包括但不限于：
  - **新建 / 编辑主题**：至少含 **主题名称**；须含管理员 **手填排序值 `sort`**（整数，用于主题列表顺序，见 §0.5.1）；简介、是否对外展示、软删除标记等实现期可定。
  - 在主题下查看 **待审留言**，对单条执行 **通过（上墙）** / **不通过（拒绝）** 等操作（具体按钮命名实现期可定）。
  - 对 **已上墙留言** 维护 **加/减权**、**置顶**（作用域见 §0.8、§0.7）。
- **鉴权**：不设用户登录体系；使用 **环境变量中配置的一枚长随机密钥** 作为管理员凭据（站主已接受）。请求管理接口时由服务端校验（例如 `Authorization: Bearer <密钥>` 或等价安全做法）；**密钥不得写入前端公开 bundle**，仅服务端与站主本地保管。
- 管理页 URL 可实现为 **不可猜测路径**（如 `/guestbook-admin-<随机片段>`）或固定路径 + 仅 API 强制密钥，二者择一在实现阶段按便利性定案（需求层：必须满足「无密钥不能改审核状态、不能改主题」）。

### 0.4 昵称与正文格式（已确认）

- **昵称**：长度 **1～32** 个字符（`trim` 后计长；字符计法与实现一致即可，通常为 Unicode 标量/JS 字符串长度，与用户直觉中的「字」一致）。
- **正文**：
  - **下限**：`trim` 后 **不少于 100** 个字符；**低于 100 字符的提交不落库、不进入待审核队列，也不需要站主审核**。
  - **上限**：**5000** 个字符（`trim` 后计长）。
  - **换行**：**允许**；展示时按换行渲染。
  - **链接（已确认）**：正文中符合规则的 **`http://` / `https://` URL** 在展示时渲染为 **可点击外链**（`target="_blank"` + `rel="noopener noreferrer"`）；**禁止** `javascript:` 等非 http(s) 方案；非 URL 文本仍按 **纯文本转义** 防 XSS。识别规则实现期可用安全 URL 正则 + 长度上限（如单链 ≤500 字符）。
- **限额计数**：仅 **已成功写入数据库** 的留言计入「同 IP 日 5 条」「全站日 1000 条」（**跨主题合计**，不因分区而单独翻倍）；因长度校验失败而未落库的提交 **不计入** 上述限额（避免无效请求误伤真实额度）。
- **交互**：不满足长度等校验时，接口返回 **machine-readable `reason`**（与 **§0.10** 对齐），前端展示 **§0.10** 对应中文提示。

### 0.5 页面与列表版式（参考知乎信息流）

#### 0.5.1 主题列表（入口页）

- **排序（已确认）**：访客可见的主题列表 **按管理员手填字段 `sort` 排序** — 采用 **`sort` 升序**（**数值越小越靠前**）。`sort` 相同则二级规则实现期定（建议 **`id` 升序** 或 **创建时间**）。
- **分页**：面向访客的 **主题列表** **每页展示 5 个主题**；超过 5 个时提供 **下一页 / 页码**（实现期二选一或并存）。
- **单主题卡片**：至少展示 **主题名称**；可选展示简介、**已上墙留言条数**（实现期定，注意控制 D1 查询次数，可冗余计数列或定时汇总）。
- **交互**：点击某一主题进入 **该主题下的留言列表页**（或同页切换视图，实现期定）。

#### 0.5.2 主题下的留言列表

- **分页**：某一主题内，公开留言列表 **每页 10 条**（**仅统计「非置顶」的已上墙条目，见 §0.7**）。
- **单卡结构（对齐知乎信息密度习惯）**：
  - **标题行**：以 **昵称** 作为主标题展示（粗体、醒目）；不单独强制用户填「标题」字段。
  - **正文区**：默认 **缩略展示**（如前 2～3 行或固定字符数，实现期定），超出部分用省略；尾端提供 **可点击的展开指示**（如「▼ / 展开全文」类）。
  - **展开 / 收起**：点击展开后显示 **全文**（保留换行）；**再次点击** 同入口可 **收起** 回缩略态。
- **可选增强**：是否显示「通过审核 / 上墙」时间等小字信息，实现期定。
- **排序切换 UI**：在 **某一主题的留言列表** 内，区域 **右上角**（或工具条右侧）提供 **两种排序方式**。访客点赞保留但设资格门槛（见 §0.7 与 §4），不提供点踩。
  - **推荐**：该主题下，非置顶区按 **推荐排序分** 降序（见 §0.7：时间衰减 + **合格访客点赞热度** + **站主加减权**；**仅在当前主题内比较**）。
  - **最新**：该主题下，非置顶区按 **发布时间** **新→旧**。

### 0.6 测验完成门槛与客户端标志

- **业务规则**：用户须 **至少成功完成过一次「标准测试」或「PRO 测试」中的任意一种**，才允许 **在某一主题下发留言**（须携带 **主题 ID**，且主题须为 **正常展示中** 状态；**已删除/已隐藏** 的主题不可再投稿，见 §0.8）。
- **PRO 上线节奏**：PRO 版 **无限期推迟**；**实现期可先只接通标准测试** 的资格校验与 `localStorage` 写入。**需求层保留**「PRO 完成亦具备资格」；数据结构与接口宜 **预留** PRO 完成态，待 PRO 上线后补足服务端校验，无需推翻留言板设计。
- **标志存储**：以 **浏览器 `localStorage` 持久化** 资格标记（可与 `basic-test-session` 成功态 **合并或分键**，实现期统一）；在对应测验 **得出有效结果** 时写入；**清除站点数据** 后需重新测验才能再互动。
- **服务端**：须在 **留言提交** 接口侧具备 **可校验的资格**（**防伪造**）；实现期优先采用 **Signed token / HttpOnly Cookie / 与落库 attempt 的弱关联** 等之一；需求层拒绝「仅前端隐藏按钮、接口完全不校验」。

### 0.7 推荐排序、置顶与分页

- **设计取舍**：保留用户端 **点赞**，但必须设置资格门槛、频率限制和去重；**不提供点踩**。点赞只允许完成普通版测试且结果为 **理智正常** 或 **满遗** 的访客使用；**萌萌人**、**旧明粉**、**新明粉**、**朱元璋梦男** 只能浏览留言板。原因：Cloudflare / D1 对读、写次数有计划内限额，点赞若不加门槛会放大写入。
- **推荐算法（仅用于「推荐」排序，已给默认实现）**：
  - **上墙时间**：`publishedAt` 为该留言 **审核通过、对公众可见** 的起算时刻（存 UTC ISO 或等价；与「最新」排序同一字段）。
  - 令 `Δh = max(0, (now - publishedAt) / 3600000)`，即距今经过的 **小时数**（毫秒差 / 3_600_000）。
  - **新鲜度**：\(F(\Delta h) = e^{-\Delta h / \tau}\)，默认 **τ = 120**（小时）。约每 **83 小时** 衰减到 **`1/e`** 权重。
  - **推荐分**（同一主题内、**非置顶** 条目）：  
    `rankScore = adminDelta + likeScore + K * F(Δh)`  
    默认 **`K = 1000`**；**`adminDelta`** 默认 **0**，建议管理端限制在 **[-500, 500]**（整数，实现期可收紧）；**`likeScore`** 仅统计符合资格门槛的有效点赞。
  - **并列规则**：`rankScore` 相同则按 **`publishedAt` 降序**（新者优先）；仍相同可用 **`id` 降序**。
  - 参数 **`τ`、`K`、adminDelta 边界** 可放环境配置，便于日后调参 **无需改需求**。
- **置顶**：
  - **按主题生效**：每个主题内 **最多置顶 3 条**（**已确认**）；**置顶区始终位于该主题留言列表最上方**。
  - **置顶** **不参与、不跟随** 「推荐 / 最新」；切换排序时 **仅该主题内非置顶区** 重排。置顶条目之间的顺序由 **管理页在该主题下的置顶顺序**（或指定字段）决定。
  - 管理端对第 4 条执行置顶时返回错误，见 **§0.10** `pin-limit-exceeded`。
- **分页与排序（已确认）**：在 **某一主题内**，对每个排序模式：
  - **置顶留言**（≤3 条）全部展示在列表 **顶部**，**不计入** 「每页 **10** 条」的额度；**每页固定最多 10 条非置顶** 已上墙留言。
  - 分页截取对象为：**置顶块（最多 3 条）+ 当前排序下非置顶列表的第 N 页 10 条**。
- **站主运营字段**（管理页，仅密钥）：对 **某主题下** 单条留言 **加/减权**（参与该主题内 **推荐** 排序）、**置顶** 开关与顺序。

### 0.8 主题（版块）与数据归属

- **模型**：留言板由多个 **主题** 组成；每条 **用户留言** 必须隶属于 **一个且仅一个** 主题（外键或等价关联）。
- **管理员**：通过管理页 **新建 / 编辑主题**（含手填 **`sort`**，§0.5.1）；持密钥可对主题做 **删除（推荐软删除）** 等操作。
- **删除主题（已确认）**：管理员删除某一主题后，该主题 **对访客不可见**（不出现在主题列表、不可进入）；其下 **已有留言（含待审、已上墙）对公众全部隐藏**。实现推荐 **软删除**（如 `deleted_at` / `is_active=0`），**不必**物理删库；管理端是否仍可在后台查看历史数据，实现期可定。
- **访客路径**：顶栏 **留言板** → **主题列表**（每页 **5** 个主题，§0.5.1）→ 进入主题 → **该主题留言**（每页 **10** 条非置顶，§0.5.2、§0.7）。
- **审核与展示**：待审留言在 **所属主题** 的管理视图中处理；用户仅在对应主题下看到 **该主题已上墙** 的留言。
- **限额**：**同 IP 日 5 条**、**全站日 1000 条** 为 **所有主题合计**（§0.4），不因分区而单独翻倍。

### 0.9 提交成功后的用户提示（已确认）

- 用户 **成功提交** 留言（已落入待审队列、通过服务端校验）后，前端 **必须** 给出明确反馈，文案需同时传达：
  1. **提交已成功**；
  2. 留言处于 **待管理员审核** 状态；
  3. **审核通过后** 才会在该 **主题** 下对所有人展示。
- **推荐示例**（可微调用字，不得省略上述三点含义）：  
  **「提交成功。您的留言已进入审核，通过后将会显示在本主题下。」**
- 若接口返回「未落库」类失败（如篇幅不足），则 **不得** 使用上述成功文案，须对应错误提示（§0.10）。

### 0.10 失败场景清单与提示文案（已确认）

接口宜返回 JSON：`{ "ok": false, "reason": "<下表 code>" }`（HTTP 4xx/5xx 依实现）；前端 **按 `reason`** 展示 **用户可见文案**（可微调语气，含义不变）。

| `reason`（建议 code） | HTTP 参考 | 用户可见文案（简体中文） |
|------------------------|-----------|---------------------------|
| `body-too-short` | 400 | 正文至少需要 100 个字，补充后再提交。 |
| `body-too-long` | 400 | 正文过长，请删减至 5000 字以内。 |
| `nickname-empty` | 400 | 请填写昵称。 |
| `nickname-invalid` | 400 | 昵称为 1～32 个字，请修改后重试。 |
| `topic-not-found` | 404 | 找不到该主题，请返回留言板首页重新选择。 |
| `topic-inactive` | 400 | 该主题已关闭，请选择其他主题留言。 |
| `not-qualified` | 403 | 请先完成一次标准测试后再留言。 |
| `qualification-invalid` | 403 | 资格校验未通过，请重新完成测试或刷新页面后再试。 |
| `ip-daily-limit` | 429 | 您今日留言次数已达上限，请明天再来。 |
| `site-daily-limit` | 429 | 本站今日留言名额已满，请明天再来。 |
| `pin-limit-exceeded` | 400 | 每个主题最多置顶 3 条，请先取消其他置顶。 |
| `invalid-json` | 400 | 提交数据异常，请刷新页面后重试。 |
| `database-not-configured` | 503 | 服务暂不可用，请稍后再试。 |
| `database-write-failed` | 503 | 提交失败，请稍后再试。 |
| `internal-error` | 500 | 服务繁忙，请稍后再试。 |

补充说明：

- **未列出的 `reason`**：统一展示 **「操作失败，请稍后再试。」**（可选 `unknown-error`）。
- **网络中断 / 无响应**：前端可用 **「网络异常，请检查连接后重试。」**

### 0.11 SEO 与收录策略（说明 + 已确认）

- **SEO 是什么**：**搜索引擎优化**（Search Engine Optimization），指通过页面结构、标题、描述、可被爬虫访问等方式，让 **百度、Google、Bing** 等 **收录并展示** 你的网页，从而带来 **自然搜索流量**。与之相对，**`noindex`** 是告诉爬虫 **不要索引当前页**，减少 **UGC 留言** 被搜索快照长期缓存带来的 **合规与舆情风险**。
- **本项目（已确认）**：留言板 **主题列表、主题内留言等 UGC 页面** 对爬虫使用 **`noindex, nofollow`**（`<meta name="robots">` 或 HTTP 头等价方式）；**站点首页、测试页** 等是否允许收录 **不与本需求冲突**，由全站策略另定。
- **不做**：为留言板单独做 **关键词堆砌**、**隐藏文字** 等黑帽 SEO。

## 2. 范围声明

### 2.1 包含（已初步对齐）

- **主题**：管理员可 **新建主题**；用户留言 **必须在某一主题下** 提交（§0.8）。
- **具备测验资格者**（标准 **或** PRO 任一完成，见 §0.6；**当前实现可先只做标准**）可 **匿名** 在主题下留言；未达标者仅浏览。
- 用户自填 **昵称** + **留言正文**（并指定/处于某一主题语境）。
- **同一 IP** 每自然日最多提交 **5** 条。
- **全站**每自然日最多接收 **1000** 条新留言（达到上限后当日不再接受提交）。

### 2.2 策略声明（非 MVP 功能，已确认不加）

- **违禁词过滤**：**不做** 自动拦截；依赖 **管理员人工审核**（§0.3）。
- **用户举报入口**：**不做**；同上。
- **推荐公式 / 失败文案 / URL / SEO / 置顶上限**：已分别在 **§0.7、§0.10、§0.4、§0.11、§0.7** 固化。
