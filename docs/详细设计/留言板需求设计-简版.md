# 留言板需求（简版）

单人开发用；细节见 `留言板需求设计.md`。

## 路由与信息架构

- 顶栏入口 **留言板** → **主题列表**（每页 **5** 个，`sort` **升序**，小在前）→ 点进主题 → **该主题留言**（非置顶每页 **10** 条；**置顶 ≤3**，置顶不占这 10 条额度，全部顶在上方）。
- 主题内排序：**推荐** | **最新**。推荐分：`rankScore = adminDelta + 1000 * exp(-Δh/120)`，`Δh` = 距 `publishedAt`（上墙时间）的小时数；并列再按 `publishedAt`、`id` 降序。

## 数据与状态

- **主题**：`title`、`sort`、`deleted_at`/`is_active` 软删；删后访客不可见，下挂留言（含待审/已上墙）**对公众全隐藏**；仍可投稿校验须拒绝已删主题。
- **留言**：挂 `topic_id`；状态：待审 / 拒绝 / 已上墙；上墙时写 `published_at`；`admin_delta`（建议 [-500,500]）；置顶顺序字段。
- **限额（北京时间自然日，跨主题合计）**：同 IP **5** 条/日、全站 **1000** 条/日；仅成功落库计数；不足 100 字正文不落库、不计额。

## 用户侧

- **发帖资格**：须完成 **标准测试**（PRO 可先不做）；服务端必须可校验，不能仅靠 localStorage。
- **字段**：昵称 1～32 字；正文 100～5000、可换行；正文内 **http/https** 渲染为外链（新窗口 + `noopener`，禁 `javascript:`）。
- **成功提示**：需体现「已提交 + 待审核 + 通过后在本主题展示」（示例：提交成功。您的留言已进入审核，通过后将会显示在本主题下。）
- **失败**：用 `reason` 映射文案（与长文 §0.10 一致）；常见：`body-too-short/long`、`nickname-invalid`、`topic-not-found/inactive`、`not-qualified`、`ip-daily-limit`、`site-daily-limit`、`database-*`、`internal-error`。
- **UI**：昵称为卡标题感；正文默认可展开/收起。

## 管理侧

- **鉴权**：`Authorization: Bearer` + 环境变量密钥；密钥不进前端 bundle。
- **能力**：新建/编辑主题（含手填 `sort`）、软删主题；按主题审帖（通过写 `published_at`）、拒绝；`admin_delta`；置顶（**最多 3** 条/主题，第 4 条返回 `pin-limit-exceeded`）。
- **不做**：违禁词机审、用户举报、点赞踩。

## 其它

- **SEO**：留言板相关页 **`noindex, nofollow`**。
- **技术栈**：现有 Next API + Cloudflare D1；注意 D1 读写配额。
