## 1. 两套题库的关系

| 产品线 | 题型 | 文档载体 |
| --- | --- | --- |
| **普通版** | 每题 **支持 / 反对**（认同题干与否），每次 **20** 题 | **[`题库/lite/`](./题库/lite/)**：段落维护（序号 / 题干 / 明粉误选各一段），按主题拆成核心 / 财政 / 军事 / 行政 / 横向史观 / 扩展；可由 `npm run export:lite-judgment-md` 生成草稿后再手工改题干 |
| **Pro 版** | 每题 **4** 选项（A–D），每次 **20** 题（核心 6 + 补充 14） | **[`题库/pro/`](./题库/pro/)** 下按类别拆分的 Markdown（含扩展题） |

**扩展题**（`pro/06-extended.md`）默认纳入 **Pro 四选一**抽题池；普通版抽题若将来纳入扩展段母题，先在 `lite/06-extended.md` 增补对应短陈述即可。

## 2. 普通版（判断题）撰写与评分口径

- **题干**：单一陈述句，一道题只测一个判断；文档维护表见 [`题库/lite/README.md`](./题库/lite/README.md)。
- **答案**：每题标注理性应选择 **支持题干** 或 **反对题干**；与 [`basic-evaluation-standard.md`](./basic-evaluation-standard.md) §1 的一维标定一致。
- **计分**：仅 **支持 / 反对** 二值；可按核心 / 补充权值加权累加为 \(S\)，再映射到与 Pro 相同的 `resultId`（见评测标准 §1.5）。
- **校对重点**：避免一眼假；避免依赖未核实极端数字；陈述语气尽量接近真实网络争论而非教科书定义句。

## 4. Pro 版选项评分口径（双轴）

每题 4 个选项。默认语气与代码评分一致：

| 选项 | 历史了解程度 | 明朝偏向程度 | 方向 |
| --- | ---: | ---: | --- |
| A | 0 | 3 | 强明粉 / 朱元璋梦男式判断 |
| B | 1 | 2 | 低了解但高明朝偏向 / 旧明粉式辩护 |
| C | 0 | 1 | 低了解、中等明朝偏向 / 教材式直觉 / 路人摇摆 |
| D | 3 | 0 | 倾向材料、机制和制度后果分析 |

---

## 5. 校对重点（Pro 四选一）

- A/B/C 是否像真实答题者会选的自然回答，而不是模板化立场标签。
- 是否有题目立场太明显，让 D 看起来像唯一正确答案。
- 是否有题目过度依赖未核实数字或具体案例。
- 军事、行政题是否足够独立，不只是财政题换皮。
- 横向史观题是否仍能服务 **共用结果档位**（见 [`basic-evaluation-standard.md`](./basic-evaluation-standard.md) §3），避免写成需要独立「四维」才能解释的题。
- 史同、粉圈、教条主义题量是否克制。
