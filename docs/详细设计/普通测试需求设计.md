# 普通测试需求设计（核心版）

单人开发 / AI 上下文用。评测语义见 `docs/详细设计/basic-evaluation-standard.md`，全局交接见 `docs/实现/handoff.md`。

## 1. 产品目标

- 明清史观向 **趣味测试**：低门槛、易传播，结果适合截图玩梗。
- **不做** 登录、**不保存** 个人逐题记录；可做 **匿名聚合统计**（如 `basic_attempts`）。
- **MVP** 不做结果海报。

## 2. 用户能做什么

- 从首页进入 **普通测试**（`/test`），答完看 **结果页**（`/result`）。
- 一次测验 **20 题**，单选题；支持进度展示、选题、全部答完后提交跳转结果。
- 结果展示：**称号、简要解释、分享文案**；可再测；可外链到站主 B 站 / 知乎（产品上还预留从结果进 Pro，Pro 另文）。

## 3. 题库与抽题（与代码一致）

- 题库分 **`core` / `supplemental`** 两池（源码：`basic-core-questions.ts`、`basic-supplemental-questions.ts`）。
- 每次测验固定 **20 题**：从核心池抽 **6** 题、扩展池抽 **14** 题，再 **洗牌** 成顺序（配置：`src/data/basic-test-config.ts`）。
- 核心题 **权值 2**，扩展题 **权值 1**；每题选项对两个维度贡献 **0～3** 的整数步长（与选项 `score` 一致）。

## 4. 计分与结果

- **两个维度**（与 `BasicScore` 一致）：
  - **`historyKnowledge`**：历史了解程度  
  - **`mingPreference`**：明朝偏向程度  
- 单维度得分 = 所选题目的「选项分 × 题目权值」之和；**理论单轴满分 78**（\(6×2×3 + 14×1×3\)）。
- **结果分档**：按两维落在 `basic-results.ts` 中配置的 **矩形区间** 匹配主档；逻辑见 `src/lib/basic-scoring.ts`。
- **「满遗」彩蛋**：基础档为「理性史观」对应项时，以配置概率随机替换为 **「满遗」** 展示名（仍为同一套理性区间逻辑，`displayChance`）。

### 4.1 产品期望的 7 类主结果（交接文档）

站主在 `docs/实现/handoff.md` 中列出的 **传播用语** 包括（与页面最终文案可能微调）：

1. 理性历史观  
2. 完全不了解历史  
3. 对历史了解不多，但轻度偏好明朝的萌萌人  
4. 已经切割的旧明粉  
5. 旧明粉  
6. 新明粉  
7. 朱元璋梦男  

另：**「满遗」** 非独立档位，为「理性历史观」的彩蛋显示名。

### 4.2 与评测标准对齐（维护时注意）

- **结果分档**已与 [`basic-evaluation-standard.md`](basic-evaluation-standard.md) §6 一致：单轴满分 **78**（每次抽 6 道 core×2 + 14 道 supplemental×1）；区间按该文档表内 **×78/72** 换算写入 `basic-results.ts`。  
- **文案、边界**仍应以 `basic-evaluation-standard.md` 与 `docs/实现/handoff.md` 持续校对；正式题干来自 [`题库.md`](题库.md) 与 `题库/` 目录，再导入 `src/data/*questions*.ts`。

## 5. 会话与匿名上报

- 测验进行中/刚结束：使用 **`sessionStorage`**（或等价）存本次抽题与作答，键名等同 `src/lib/basic-test-session.ts` 约定。
- 结果页可调用 **`POST /api/basic-attempts`** 上报匿名汇总（含结果 id、两维分数、耗时等）；无 D1 或失败时**静默**均可（见既有 `route.ts` 行为）。

## 6. 内容与合规

- 题目可讽刺，但全站要有 **娱乐向免责**（公告/关于），见 `docs/实现/announcement-draft.md` 与 `docs/实现/handoff.md`。
- **勿**把论文、知乎原文大段粘进 **对外** 页面；参考资料放 `docs/references/`。
