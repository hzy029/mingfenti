| 字段 | 内容 |

## 1. 背景与约束

项目面向公开互联网访问，核心风险不是传统意义上的复杂账号体系攻击，而是：

- 管理员接口被猜到或误调用，导致主题、回答、评论被篡改。
- 访客接口被脚本反复请求，消耗 Cloudflare 免费套餐的请求次数、CPU 时间、D1 读写额度。
- 点赞、留言、统计上报这类轻交互接口被刷，造成数据库写入膨胀。
- UGC 内容引入 XSS、违法违规内容、垃圾外链等展示风险。

当前策略应优先选择“便宜、简单、服务端可执行”的防线：先在 Cloudflare 边缘和 API 层挡掉明显异常请求，再减少每个正常请求触发的 D1 查询和写入。

## 2. 安全目标

### 2.1 管理员认证

- 管理员只能通过 `/admin/board` 进入管理页。
- 管理 API 必须校验服务端环境变量 `ADMIN_BOARD_SECRET`。
- 密钥不得写入 Git、不得放入前端公开 bundle、不得在 URL 查询参数中传递。
- 未配置密钥时，管理 API 应默认不可用，而不是降级为开放。

### 2.2 防压力请求

- 限制单 IP、单路径、单动作的高频请求。
- 对写入型接口设置更严格的频率和每日额度。
- 对列表读取接口设置缓存、分页和字段收敛，减少 D1 读取。
- 对非法请求尽早返回，不进入 D1。

### 2.3 D1 配额保护

- 能不写库的请求不写库，例如无效表单、过短测试、格式错误 JSON。
- 能缓存的列表不实时查库。
- 能聚合的统计不重复扫描全表。
- 高频互动功能必须设置资格门槛、频率限制和去重，避免把免费额度花在低价值写入上。

### 2.4 UGC 展示安全

- 用户输入只按纯文本或受控 Markdown 展示。
- 链接只允许 `http://` / `https://`。
- 管理员保留隐藏、置顶、调权等治理能力。
- 公开 UGC 页面建议 `noindex`，降低搜索快照长期留存风险。

## 3. 当前已实现

### 3.1 管理员密钥

当前管理端采用单密钥认证：

- 管理页：`/admin/board`
- 服务端环境变量：`ADMIN_BOARD_SECRET`
- 请求头：`x-admin-board-secret`
- 校验逻辑：`src/lib/board-admin-auth.ts`

行为约定：

| 场景 | 返回 |
| --- | --- |
| 未配置 `ADMIN_BOARD_SECRET` | `503`，`reason: "admin-not-configured"` |
| 密钥错误或缺失 | `401`，`reason: "unauthorized"` |
| 密钥正确 | 允许访问管理 API |

现有校验已使用定长字符串比较逻辑，避免简单的逐字符提前返回。密钥部署、轮换和故障处理见 [ADMIN_BOARD_SECRET-配置说明](../实现/ADMIN_BOARD_SECRET-配置说明.md)。

### 3.2 管理入口弱隐藏

- 顶栏不挂 `/admin/board`。
- 管理页需要站主自行收藏 URL。
- 这只能减少普通用户误入，不是安全边界；真正边界仍是服务端密钥。

### 3.3 测试统计写入保护

`POST /api/basic-attempts` 已做基础保护：

- JSON 解析失败直接 `400`。
- payload 字段不完整直接 `400`。
- `durationSeconds < 30` 时返回成功但不写库。
- D1 未配置或写入失败时静默降级，不影响用户看结果。

这能减少刷新、误触、极快脚本提交造成的无意义统计写入。

### 3.4 UGC 输入长度控制

留言板文本入口已做裁剪：

| 字段 | 当前上限 |
| --- | --- |
| 主题标题 | 200 |
| 回答正文 | 8000 |
| 评论正文 | 4000 |
| 昵称 | 64 |

无正文的回答和评论不会写入数据库。

### 3.5 D1 不可用降级

多个接口在 D1 未绑定时返回 `database-not-configured`，避免本地开发或部署配置错误时抛出不可控异常。

## 4. 需要补齐的核心模块

### 4.1 Cloudflare 边缘防护

优先在 Cloudflare 控制台配置，挡在 Worker 之前，减少 Worker 请求数和 CPU 消耗。

建议规则：

| 类型 | 建议 |
| --- | --- |
| WAF / 安全规则 | 对 `/api/*`、`/admin/*` 单独建规则，拦截明显异常 UA、空 UA、异常国家或已知恶意来源 |
| Rate Limiting | 对写入接口配置更低频率；对管理接口配置更严格频率 |
| Bot Fight / 托管质询 | 对可疑请求启用 JS Challenge 或 Managed Challenge |
| 缓存规则 | 对静态资源、公开只读页面尽量走边缘缓存 |
| 告警 | 关注请求数、CPU 时间、D1 rows read / written 的异常峰值 |

不要在文档中写死 Cloudflare 免费套餐的具体额度；额度可能调整，以 Cloudflare 控制台和官方当前限制为准。

### 4.2 服务端限流

边缘防护不能覆盖所有情况，API 层仍需要最小限流。建议新增通用限流模块，例如 `src/lib/rate-limit.ts`，按 IP、路径、动作和日期生成 key。

最小规则：

| 动作 | 建议限制 |
| --- | --- |
| 测试统计上报 | 单 IP 每分钟 3 次，单 IP 每日 20 次 |
| 发回答 | 单 IP 每小时 3 次，单 IP 每日 5 次，全站每日 1000 次以内 |
| 发评论 | 单 IP 每小时 10 次，单 IP 每日 30 次，全站每日 3000 次以内 |
| 点赞 | 仅允许已完成测验且结果在允许档位的访客；单 IP 每分钟 20 次，单 IP 每对象每天 1 次 |
| 管理 API | 单 IP 每分钟 10 次；连续失败 5 次后短暂封禁 |

实现选型：

- 短期：用 Cloudflare WAF / Rate Limiting 先挡高频请求。
- 中期：用 D1 表记录日额度和失败次数，但要注意限流本身也会写库。
- 更优：如果后续可用 KV / Durable Object，再把高频限流计数迁到更适合的存储。
- IP 计数可存储“日期盐 + IP”的哈希值，不保存明文 IP；日期盐按北京时间自然日轮换，满足日额度统计即可。

### 4.3 写入型接口前置校验

所有 `POST` / `PATCH` 接口应遵循同一顺序：

1. 校验请求方法和 `Content-Type`。
2. 限制 body 大小。
3. 解析 JSON。
4. 校验字段类型、长度、枚举值。
5. 执行 IP / 动作限流。
6. 必要时再读取 D1。
7. 最后写入 D1。

这样可以让大量垃圾请求在进入数据库前就结束。

### 4.4 点赞资格与降成本

当前回答和评论点赞接口每次点击至少涉及查存在、更新、再查新热度，容易成为刷写入口。

已确认规则：

- 访客必须先完成**一次有效测验**（普通版判断或 Pro 四选一，与 [`basic-evaluation-standard.md`](../详细设计/basic-evaluation-standard.md) 一致；共用 `resultId`）。
- 仅当结果为 **客观中立**（`objective-neutral`）或 **满遗**（`manchu-loyalist`）时可点赞（与实现白名单一致时可微调）。
- 结果为 **萌萌人**、**旧明粉**、**新明粉**、**朱元璋梦男** 的访客只能浏览留言板，不得点赞。
- 点赞资格只影响点赞，不等同于管理员权限。
- 点踩不做。

资格校验建议：

1. 前端根据测验结果控制按钮可见性和提示文案。
2. 服务端仍必须校验资格凭据，不能只信前端。
3. 资格凭据应能证明「完成过测验」和「结果类型」，避免用户随便改 `localStorage`。
4. 每个回答/评论对同一访客或同一 IP 哈希每天只允许点赞一次。

实现方案对比：

| 方案 | 成本 | 说明 |
| --- | --- | --- |
| 前端本地资格判断 | 低 | 体验友好，但不能作为安全边界 |
| 服务端签名测试结果 | 中 | 完成测试后发放短小签名 token，点赞时校验；适合当前项目 |
| 服务端存测试完成记录 | 中 | 用 D1 查询资格，可信但增加读写 |
| IP + 对象去重 | 中 | 需要新增表或 KV，能防一部分刷赞 |
| 批量聚合点赞 | 较高 | 用队列或定时聚合，当前项目不必优先 |

在免费套餐下，建议优先采用“服务端签名测试结果 + IP 哈希去重 + 分钟级限流”的方案。这样既保留点赞玩法，也不会把 D1 读写完全暴露给脚本。

### 4.5 读取缓存与分页

公开读取接口应优先控制 D1 读取：

- 主题列表固定分页，例如首页只取少量热门主题。
- 主题详情回答列表分页，不一次性读取全部回答。
- 评论按回答分页或点击后加载。
- 首页热门模块可短缓存 30 到 300 秒。
- 不为每个卡片实时查询复杂统计；需要时使用冗余计数字段或定时汇总。

### 4.6 管理端失败保护

管理接口密钥错误时要避免被暴力猜测：

- 连续失败达到阈值后，对该 IP 暂停响应一段时间。
- 管理接口错误信息保持简洁，不返回服务端配置细节之外的信息。
- 密钥轮换后，旧 sessionStorage 中的密钥自然失效。

### 4.7 DeepSeek 云端 Agent 审核

回答和评论已按本节方向接入 DeepSeek API 审核链路，由云端 Agent 做发布前审核；真实密钥仍只通过 Cloudflare Secret / `.env.local` 配置，不写入仓库。

已确认策略：

- **审核模式**：同步审核。用户提交后，服务端先完成本地校验，再立即调用 DeepSeek。
- **审核尺度**：保守通过。只有明确合规、非辱骂、非广告、非恶意外链、非刷屏的正常讨论才自动公开。
- **默认模型**：`deepseek-v4-flash`。后续可通过 `DEEPSEEK_MODEL` 覆盖。
- **失败兜底**：调用失败、超时、返回非 JSON、额度异常、模型不确定时，全部进入待人工审核，不自动公开。
- **Turnstile**：仍为 P2，等访问量上来再接入。

环境变量：

| 变量名 | 必填 | 用途 | 安全要求 |
| --- | --- | --- | --- |
| `DEEPSEEK_API_KEY` | 是 | DeepSeek API 访问密钥 | 只能放 Cloudflare Secret 或本地 `.env.local`；不得写入 Git、`wrangler.toml`、日志、前端 bundle |
| `DEEPSEEK_MODEL` | 否 | 覆盖默认审核模型 | 默认 `deepseek-v4-flash`；可留空 |

状态流：

1. 用户提交回答或评论。
2. 服务端完成格式校验、资格校验、限流、额度判断。
3. 服务端调用 DeepSeek API 做同步审核。
4. DeepSeek 返回 `approve`：内容写入或更新为 `status = "published"`，同时写入 `published_at`。
5. DeepSeek 返回 `review`，或调用失败、超时、非 JSON、额度异常：内容进入 `status = "pending"`，只在管理端待审列表可见。
6. DeepSeek 返回 `reject`：内容进入 `status = "rejected"` 或待人工复核，默认不公开。
7. 管理员可覆盖 DeepSeek 结果：通过、隐藏、拒绝、退回待审。

审核结论建议：

| 结论 | 含义 | 公开策略 |
| --- | --- | --- |
| `approve` | 明确合规、非辱骂、非广告、非恶意外链、非刷屏 | 自动公开 |
| `review` | 模型不确定、争议强、疑似违规、调用失败、超时、返回非 JSON、额度异常 | 待人工审核，不公开 |
| `reject` | 明显违法、威胁骚扰、垃圾广告、恶意刷屏、恶意外链 | 不公开，可进入管理员复核 |

记录与隐私：

- 不建议长期保存 DeepSeek 原始完整响应。
- 后续实现只保存 `verdict`、`labels`、简短 `reason`、审核时间、审核模型或 provider 标识。
- 不在错误响应中暴露 DeepSeek API 细节、密钥状态或完整模型返回。
- 对话中曾出现过 DeepSeek API key，真实上线前应在 DeepSeek 控制台轮换，并使用新 key 配置 Secret。

## 5. 推荐优先级

### P0：上线前必须

- Cloudflare 控制台给 `/api/*` 和 `/admin/*` 配基础限频。
- 管理密钥 `ADMIN_BOARD_SECRET` 已配置，且确认没有提交到 Git。
- 写入接口增加服务端限流，至少覆盖发回答、发评论、点赞、测试统计。
- 公开写入接口先校验 body 长度和字段，再访问 D1。
- 点赞接口必须校验测验结果资格：仅 **客观中立**、**满遗** 可点赞（展示名以 [`basic-results.ts`](../../src/data/basic-results.ts) 为准）。
- 回答和评论应进入待审状态，未审核通过前不公开展示。

### P1：稳定运行后补齐

- 建立统一 `rate-limit` 工具，集中维护规则。
- 增加全站日写入额度保护，返回 `site-daily-limit`。
- 增加 IP 日额度保护，返回 `ip-daily-limit`。
- 为首页、主题列表、热门回答做短缓存。
- 管理端连续认证失败临时封禁。
- 接入 DeepSeek API 云端 Agent，以“同步审核 + 保守通过”的方式自动审核回答和评论。

### P2：有流量后再做

- 管理员登录从单密钥升级为账号 + 密码 + 二次确认。
- 引入 Turnstile，对发帖、评论、异常点赞请求做人机验证；当前阶段暂缓，等访问量上来再加。
- 细化审计日志，记录管理操作、隐藏、置顶、调权、密钥轮换时间。
- 接入更完整的告警和用量看板。

## 6. 接口级策略草案

| 接口 | 风险 | 策略 |
| --- | --- | --- |
| `POST /api/basic-attempts` | 刷统计写入 | 保留 30 秒过滤；增加 IP 分钟/日限流；失败不影响看结果 |
| `GET /api/board/topics` | 高频列表读取 | 缓存短时间结果；固定返回字段和数量 |
| `POST /api/board/topics/:id/posts` | 垃圾回答、写库消耗 | body 长度、IP 日限额、全站日限额、默认待审、DeepSeek Agent 审核后公开 |
| `POST /api/board/posts/:id/comments` | 垃圾评论、写库消耗 | 同回答，但额度可略高；默认待审 |
| `POST /api/board/posts/:id/like` | 刷写、刷热度 | 校验测验结果；仅客观中立/满遗可点赞；IP 哈希去重和限流 |
| `POST /api/board/comments/:id/like` | 刷写、刷热度 | 同上 |
| `/api/admin/board/**` | 管理操作被撞库 | 密钥认证、失败限流、Cloudflare 管理路径限频 |

## 8. 验收标准

- 未配置 `ADMIN_BOARD_SECRET` 时，所有管理 API 均不可用。
- 错误密钥访问管理 API 返回 401，且连续失败会被限流。
- 高频请求写入接口时，服务端返回 429，不继续写 D1。
- 无效 JSON、超长正文、空正文不会写库。
- 未完成有效测验，或结果不可点赞时，点赞接口拒绝写入。
- 回答和评论未通过审核前，不出现在公开留言板。
- Cloudflare 控制台能看到 `/api/*` 请求峰值被限频规则削平。
- D1 rows read / written 没有因为单个 IP 或单个路径异常飙升。
